# Comparison dashboard

A single-page control panel for the showcase: pick an implementation, see its stack and code
metrics, **call it live** through an embedded Swagger UI, and read the code-ceremony charts.
It's the visual front door to [`docs/comparison.md`](../docs/comparison.md).

![what it shows](https://img.shields.io/badge/shows-charts%20%2B%20live%20API%20explorer-2a78d6)

## What it shows

- **Implementation switcher** — chips grouped by runtime (.NET / Python / Node); selecting one
  updates the detail panel (stack, code metrics, concurrency strategy, port).
- **Live API explorer** — embedded Swagger UI, loaded with the shared `contract/openapi.yaml`
  and pointed at the selected implementation through a same-origin reverse proxy (`/impl/<name>/`),
  so "try it out" works without CORS. Needs the implementation running.
- **Axis-A chart** — code ceremony across the .NET styles (files / lines / non-blank), the headline.
- **Axis-B chart** — code size by runtime (context) + a performance panel that fills once the
  benchmarks run.
- **Data table** — every implementation's metrics.

It reads [`data.json`](data.json) (generated by [`scripts/gen-dashboard-data.mjs`](../scripts/gen-dashboard-data.mjs));
an embedded fallback keeps the charts working even when opened directly as a file.

## Run it

The dashboard alone (charts work; the live explorer needs implementations reachable):

```bash
# from the repo root (the build copies the shared contract)
docker build -f dashboard/Dockerfile -t example-api-dashboard .
docker run --rm -p 8080:80 example-api-dashboard   # http://localhost:8080
```

Everything together — the dashboard **and all ten implementations**, each with its own database:

```bash
# from the repo root
node scripts/gen-dashboard-data.mjs          # refresh metrics (optional)
docker compose -f docker-compose.all.yml up --build
# dashboard: http://localhost:8080   ·   implementations: http://localhost:8081..8090
```

Then open the dashboard, pick an implementation, and hit **Open live API explorer**.

## How the proxy works

`nginx.conf` proxies `/impl/<name>/<path>` to `http://<name>:8080/<path>` using Docker's embedded
DNS, so the browser only ever talks to the dashboard origin. Swagger UI's `requestInterceptor`
rewrites each request to `/impl/<selected>/…` before it's sent. Add an implementation to
`docker-compose.all.yml` on the `web` network and it's reachable — no dashboard change needed.

> **Status:** the dashboard (charts + switcher) is complete. The live explorer and
> `docker-compose.all.yml` were authored without a running Docker daemon in the build
> environment — validate with `docker compose -f docker-compose.all.yml config` and a smoke run
> before relying on them.
